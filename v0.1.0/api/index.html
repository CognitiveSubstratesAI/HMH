API Reference

All names below are exported by HMH. The algebra they build on (HV, bind, cleanup, factorize, random_hv, random_codebook, Codebook, …) lives in FactorVSA; use using FactorVSA, HMH.

R-HMH — episodic memory (§8)

HMH.RoleBook — Type

A registry of named role atoms (deterministic per name once created; seed externally).

source
HMH.role! — Function

Get (or lazily create) the role atom named name.

source
HMH.Episode — Type
Episode

A typed factor graph (paper §8.2, Eq 68):

  • schema — schema / event-type code h_σ(E)
  • slots — role => (type, filler) (role r_s, ontology type τ, filler h_{f_s})
  • relations — (relrole, role_i, role_j) typed relation between two slots' fillers
  • anchors — anchorrole => payload (links to external evidence)
source
HMH.encode_episode — Function
encode_episode(E, rb) → H_E

R-HMH episode code (Eq 69): Normalize( r_schema⊗h_σ ⊕ Σ_s m_τ(s)⊗r_s⊗h_{f_s} ⊕ Σ_{(ρ,i,j)} r_ρ⊗r_i⊗r_j⊗(h_{f_i}⊗h_{f_j}) ⊕ Σ_a r_anchor(a)⊗h_a ).

source
HMH.recover_slot — Function
recover_slot(H_E, role, type, rb, codebook) → HV

Role-masked unbind of slot role (type type) from H_E, then hard-cleanup against the slot's codebook (other slots/relations act as bounded crosstalk). BipolarMAP: bind is self-inverse, so bind(m_τ, bind(r_s, H_E)) exposes the filler.

source
HMH.complete_slot — Function
complete_slot(H_E, role, type, rb, factor_codebooks) → (factors, score)

When a slot filler is itself a product h_{f_s} ≈ c_{s,1}⊗…⊗c_{s,F} (Eq 72), recover the noisy filler then run the resonator INSIDE the episode (Eq 73) to recover its factors.

source
HMH.consolidate — Function
consolidate(episodes, rb; weights=nothing) → H_template

Episodic-semantic consolidation (Eq 77): Normalize(Σ_i w_i H_{E_i}). Episodes sharing rb (the same role atoms) are aligned by construction; a weighted bundle of their codes forms a schema/script/template. Slots whose filler RECURS reinforce in the template; idiosyncratic fillers average out (recoverable common slots = schema formation).

IMMUTABLE-CODEBOOK CONTRACT: the template is a NEW immutable HV (a fresh arena handle). Consolidation does NOT mutate or grow any codebook. To make templates into a cleanup dictionary, build a NEW immutable codebook from them (a new CodebookRef).

source
HMH.episode_recovery_rate — Function
episode_recovery_rate(D, k_slot, k_rel, M; trials, rng) → mean slot-recovery accuracy

Build random episodes with k_slot typed slots (each filler an atom from its own size-M codebook) and k_rel random typed relations; encode (Eq 69); recover every slot by role-masked cleanup; return the mean fraction recovered correctly. The gate sweeps D, k_slot, and k_rel (the last INDEPENDENTLY — relations are heavier crosstalk).

source

ColBaC-HDC — representation layer (§9)

HMH.Column — Type
Column

A ColBaC column's active motifs (§9.3), each triple-tagged: microcolumn type u ∈ {:K,:L,:B}, shell tier s ∈ 0:3 (0 = hard kernel), local role a, and the motif hypervector.

source
HMH.encode_column — Function
encode_column(C, rb) → H_m

Column hypervector (Eq 84): Normalize( Σ_{u,s,a} r_u ⊗ r_s ⊗ r_a ⊗ c_{m,u,s,a} ). Same unbind-and-cleanup machinery as the R-HMH encoder reads any tagged piece.

source
HMH.recover_motif — Function

Recover a tagged motif from a column code (role-masked unbind + cleanup), like recover_slot.

source
HMH.support_code — Function
support_code(encoded, support, rb) → H_S

Active-support code (Eq 85): Normalize( Σ_{m∈S} r_m ⊗ H_m ) over the encoded columns in support (encoded[m] is column m's H_m). Used for support retrieval / context memory.

source
HMH.certificate — Function
certificate(channels, H_m, rb) → Z_m

Structured certificate hypervector (Eq 86-87): r_signature⊗H_m ⊕ Σ_ch r_ch⊗z_ch over the certificate channels (:shared, :specific, :saturation, :demotion, …). Each channel is recovered by unbinding r_ch and cleaning against that channel's codebook.

source
HMH.cleanup_margin_of — Function

Cleanup margin Δ (Lemma 1): ⟨c⁺,z⟩ − max_{c≠c⁺}⟨c,z⟩, normalized by D. Higher = cleaner.

source
HMH.confusability — Function

Confusability M_conf (Eq 54): #{c : ⟨c,z⟩ ≥ ⟨c⁺,z⟩ − γ·D}. 1 = unambiguous; higher = more confusable.

source
HMH.reuse_score — Function

Reuse signal: max |cosine| of motif m against others (cross-column shared-abstraction mass).

source
HMH.promote_signal — Function
promote_signal(reuse, margin, m_conf) → Float64

HDC side of the shell-hygiene promote rule (Eq 89): monotone ↑ in reuse and cleanup margin, ↓ in confusability. The LEARNER combines this with its factorization defect Δ_fact to decide promotion/demotion — that combination is out of scope here (NGC line).
